# AI 媒体生成 — 技能索引

## 使用说明

**每次处理媒体生成请求时，按以下顺序执行：**

```
步骤 1 → 检查 Token（AUTH.md）
步骤 2 → 识别用户意图（本文件）
步骤 3 → 调用对应 Skill（见下表）
```

---

## 授权前置检查

> **所有 Skill 调用前必须先执行授权检查**

```
读取 AUTH.md → 调用 get_api_key()
  ├── 返回有效 Token → 继续执行对应 Skill
  └── 返回 None      → 执行 AUTH.md 授权流程，完成后再继续
```

## 技能路由表

### 📤 Skill 0：图片上传（File Upload）

**文件：** `skills/upload/SKILL.md`

**触发关键词：**
- "上传图片"、"上传文件"、"传图片"
- "帮我上传"、"把这张图上传"
- "upload image"、"upload file"

**适用场景：**
- 用户提供了本地图片路径，需要先上传获取临时 URL
- 在任何图像/视频生成前，需要先上传素材到服务器
- 任何需要将本地文件转为远程 URL 的场景

**输入：** 本地文件路径（必填）

### 🎨 Skill 1-4：图像生成 / 编辑

#### 模型选择矩阵

当用户需要生成或编辑图片时，根据以下规则匹配合适模型：

| 模型 | 文件夹 | 定位 | 适用场景 |
|------|--------|------|----------|
| **Gemini** | `image/gemini-edit/` | 图像生成 + 编辑（综合） | 文生图、图编辑、背景替换、风格迁移、图像增强 |
| **Nano Banana Pro** | `image/nano-banana-pro/` | 高质量图像生成 | 商业级出图、精细构图、高画质需求 |
| **Nano Banana** | `image/nano-banana/` | 标准图像生成 | 日常配图、快速出图、批量生成 |
| **gpt-image-2** | `image/gpt-image-2/` | 照片级写实 / 艺术风格 | 照片级写实、特定艺术风格、精准构图 |

**选择逻辑：**

```
用户请求生图
  ├── 有参考图片 + 编辑指令 ──→ Gemini（图像编辑）
  ├── 指定了某个模型       ──→ 对应模型
  ├── 追求高画质 / 商业级   ──→ Nano Banana Pro
  ├── 日常快速出图          ──→ Nano Banana
  ├── 照片级写实 / 特定风格  ──→ gpt-image-2
  └── 未明确偏好            ──→ Nano Banana（默认）
```


## 意图识别决策树

```
用户发出媒体生成请求
       │
       ▼
  需要上传本地文件？
  ├── 是 ──→ [Skill 0] 文件上传 → 获取 tmp_url 后再判断后续操作
  └── 否 ──→ 任务类型是什么？
              ├── 📷 图片相关 ──→ 见图片选择逻辑
              ├── 🎬 视频相关 ──→ 见视频选择逻辑
              └── ❓ 不确定   ──→ 询问用户意图
```

## 多轮对话处理

当用户对已生成的内容进行追加修改时：

```
"再把背景换成森林" / "重新生成一张" / "换个风格"
        │
        ▼
  判断是否需要上一张图片作为输入
  ├── 需要 → 提示用户重新上传（或使用上次保存的输出）
  └── 不需要 → 使用上次的参数，修改对应部分
```

## 错误统一处理

所有 Skill 遇到以下错误时，统一按此处理：

| HTTP 状态码 | 处理方式 |
|-------------|----------|
| 401 | 清除 Token，触发 `AUTH.md` 重新授权 |
| 403 | 提示用户检查账户权限或套餐 |
| 400 | 检查参数格式，尝试修正后重试 |
| 429 | 等待 5 秒后提示用户稍后重试 |
| 500 | 提示服务端异常，建议重试 |
| 超时 | 提示网络问题，建议检查连接 |
